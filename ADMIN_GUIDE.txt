1. CUSTOMER MANAGEMENT
================================================================================
Manages all user accounts, profile details, delivery coordinates, and roles.

* CREATE (Add New Customer):
  - Go to: Admin -> Customers -> click "Add Customer" button.
  - Fill in the form fields:
    * Full Name: John Doe
    * Email Address: john.doe@example.com
    * Phone Number: +919876543210
    * Password: SecurePass123!
    * Address Line 1: Flat 405, Green Valley Apartments
    * Area / Landmark: Opposite High School Playground, Sector 4, HSR Layout
    * Location Link: https://maps.app.goo.gl/9xZ1y2w3v4u5t
    * User Role: customer (Dropdown options: customer, admin, driver)
    * Select Plan: Protein Rich Weekly (Dropdown)
    * Subscription Start Date: 2026-06-15
  - Click "Submit" to save.

* READ (View & Search Customers):
  - Go to: Admin -> Customers.
  - Features:
    * Search Bar: Type "John Doe" or "john.doe@example.com" to filter lists instantly.
    * Pagination: Use bottom navigation buttons (1, 2, Next) to flip pages.
    * View details: Click on a customer row to view full address history and plan status.

* UPDATE (Edit Customer Profile & Roles):
  - Go to: Admin -> Customers -> Click "Edit" (pencil icon) next to the user row.
  - Operations:
    * Modify Details: Edit "Address Line 1" to "Flat 502, Green Valley Apartments".
    * Toggle Role: Change role from "customer" to "admin" to grant portal access.
    * Manage Active Status: Set "Status" toggle to "Inactive" to suspend account.
  - Click "Save Changes" to commit.

* DELETE (Delete Customer Profile):
  - Go to: Admin -> Customers -> Click "Delete" (trash icon) next to the target customer.
  - Confirmation: Click "Confirm" on the pop-up warning dialog.
  - Result: Deletes user profile, logs, and billing history.

2. MENU MANAGEMENT (DISHES)
================================================================================
Used to manage the available recipe catalog prepared by the kitchen staff.

* CREATE (Add New Dish):
  - Go to: Admin -> Menu Management -> click "Add Dish" button.
  - Fill in the form fields:
    * Dish Name: Grilled Garlic-Herb Chicken Breast
    * Subscription Tier: Protein Rich (Select from Tier dropdown)
    * Meal Slot: Select multiple: [Lunch, Dinner]
    * Diet Type: Non-Veg (Select Veg or Non-Veg)
    * Description: Chicken breast marinated in herbs, grilled and served with broccoli.
    * Calories: 420 (Numeric value in kcal)
    * Dish Image: Upload a file (e.g., chicken_breast.jpg, max 5MB)
  - Click "Submit" to save.

* READ (View & Filter Recipe Directory):
  - Go to: Admin -> Menu Management.
  - Features:
    * Filter by Tier: Select "Protein Rich" dropdown to show matching dishes.
    * Filter by Slot: Check "Lunch" slot checkbox.
    * Search: Search by name (e.g. "Garlic").

* UPDATE (Edit Dish Details):
  - Go to: Admin -> Menu Management -> Click "Edit" next to the dish.
  - Operations:
    * Edit fields: Change "Calories" from 420 to 450.
    * Replace Image: Upload a fresh picture.
  - Click "Save" to save.

* DELETE (Delete Dish):
  - Go to: Admin -> Menu Management -> Click "Delete" next to the target dish.
  - Confirmation: Click "Confirm Delete" on the warning dialog.

3. SUBSCRIPTION TIERS
================================================================================
Defines subscription levels (e.g. Classic, Premium, Protein Rich).

* CREATE (Add Subscription Tier):
  - Go to: Admin -> Settings -> Subscription Tiers -> click "Add Tier".
  - Fill in the form fields:
    * Tier Name: Premium Box
    * Description: Gourmet organic meals cooked by certified dietitians.
    * Weekly Delivery Charge: 60 (Base delivery cost in rupees per week)
    * Monthly Delivery Charge: 200 (Base delivery cost in rupees per month)
    * Status: Active (Checked)
    * Is Featured: True (Checked to highlight on landing page)
  - Click "Save" to register the tier.

* READ (View Tiers):
  - Go to: Admin -> Settings -> Subscription Tiers.
  - Lists name, base delivery charges, visibility indicators, and pricing rules.

* UPDATE (Edit Subscription Tier details):
  - Go to: Admin -> Settings -> Subscription Tiers -> Click "Edit Tier" next to "Premium Box".
  - Operations:
    * Change "Weekly Delivery Charge" to 80.
    * Toggle "Is Featured" to False.
  - Click "Save" to apply.

* DELETE (Delete Subscription Tier):
  - Go to: Admin -> Settings -> Subscription Tiers -> Click "Delete" next to the tier.
  - Note: Will block deletion if active customers are subscribed to this tier.

4. TIER PRICING RULES
================================================================================
Specifies the base cost of meals inside a tier depending on diet types.

* CREATE (Add Pricing Rule to Tier):
  - Go to: Admin -> Settings -> Tiers -> click "Manage Price Rules" next to "Classic Box" -> click "Add Pricing Rule".
  - Fill in the form fields:
    * Diet Type: Veg (Dropdown: Veg, Non-Veg)
    * Price Per Meal: 120 (Price in rupees)
    * Effective From: 2026-06-08 (Date)
  - Click "Add Rule" to confirm.

* READ (View Pricing Rules):
  - Go to: Admin -> Settings -> Tiers -> Click "Manage Price Rules". Displays a list of historic and active price points.

* UPDATE (Edit Pricing Rule):
  - Go to: Admin -> Settings -> Pricing Rules -> Click "Edit" (pencil icon) next to the target rule.
  - Change "Price Per Meal" from 120 to 135.
  - Click "Update" to confirm.

* DELETE (Delete Pricing Rule):
  - Go to: Admin -> Settings -> Pricing Rules -> Click "Delete" (trash icon).
  - Confirm deletion on pop-up.

5. PLAN COMBINATION MATRIX
================================================================================
Defines custom price overrides for specific slots and durations.

* CREATE (Add Custom Plan Combination):
  - Go to: Admin -> Settings -> Pricing Matrix -> click "Add Combination Override".
  - Fill in the form fields:
    * Tier: Classic Box (Dropdown)
    * Diet Type: Non-Veg (Dropdown)
    * Duration: weekly (Dropdown options: weekly, monthly)
    * Slot Combo: breakfast_only (Dropdown options: breakfast_only, dinner_only, both)
    * Price Override: 899 (Custom package cost in rupees)
    * Delivery Charge Override: 50 (Custom package delivery cost in rupees)
  - Click "Create Override" to save.

* READ (View Pricing Matrix):
  - Go to: Admin -> Settings -> Pricing Matrix.
  - Shows custom package pricings that bypass base calculated pricing.

* UPDATE (Modify Override Values):
  - Go to: Admin -> Settings -> Pricing Matrix -> Click "Edit Override" next to "Classic-Non-Veg-Weekly-Breakfast".
  - Change "Price Override" to 849.
  - Click "Save" to save.

* DELETE (Delete Custom Override):
  - Go to: Admin -> Settings -> Pricing Matrix -> Click "Delete Override" (trash icon) next to override row.
  - System reverts calculation to base formula: (Price Per Meal * Meals) + Delivery Fee.

7. SKIP CREDITS & MANUAL OVERRIDES
================================================================================
Oversees meal skipping, manual adjustment, and refund credit approvals.

* CREATE (Log a Manual Skip Request):
  - Go to: Admin -> Skip Override Dashboard -> click "Log Manual Skip".
  - Fill in the form fields:
    * Customer: John Doe (ID: 12) (Searchable dropdown)
    * Skip Date: 2026-06-16 (Target skip date)
    * Session Slot: Breakfast (Select Breakfast, Dinner or Both)
    * Support Notes: Customer out of town; called helpdesk.
  - Click "Submit Skip Override" to register.

* READ (Review Pending Credits):
  - Go to: Admin -> Skip Overrides -> click "Pending Approvals" tab.
  - Displays list of skips requiring administrative sign-off before credit points are rewarded.

* UPDATE (Approve/Reject Skip Credit):
  - Go to: Admin -> Skip Overrides -> click "Review" (pencil) next to John Doe's skip request.
  - Actions:
    * Override Status: Select "Approved" (Dropdown)
    * Reviewer Notes: Approved medical exemption.
  - Click "Confirm Status". This issues a wallet credit point to the user.

* DELETE (Cancel Skip Request):
  - Go to: Admin -> Skip Overrides -> Click "Cancel Skip" next to the pending request.
  - Restores the customer's delivery schedule back to active for that date.
